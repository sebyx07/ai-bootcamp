# Module 04: Ruby on Rails

Rails is a web framework that lets you build full web applications fast. It follows "convention over configuration" -- if you follow its patterns, Rails does a lot of work for you. By the end of this module, you will build a complete blog application.

## What you will learn

- How to create a new Rails application
- The MVC pattern (Models, Views, Controllers)
- How Rails routes map URLs to controller actions
- Database models and associations
- Validations, forms, and strong parameters
- How to build a complete CRUD application

## Challenges

| # | Challenge | What you will do |
|---|-----------|-----------------|
| 1 | New Rails App | Create and explore a fresh Rails application |
| 2 | Scaffold and Explore | Use Rails generators and understand what they create |
| 3 | Models | Create models and work with the database |
| 4 | Controllers | Write controller actions that handle requests |
| 5 | Views and ERB | Build HTML views with embedded Ruby |
| 6 | Routes | Define and understand Rails routing |
| 7 | Associations | Connect models together (has_many, belongs_to) |
| 8 | Validations | Add rules to your data (required fields, length limits) |
| 9 | Forms and Strong Params | Build forms and handle form data securely |
| 10 | Build a Blog | Put it all together -- build a complete blog app |

## Why this matters

Rails powers some of the biggest sites on the web (GitHub, Shopify, Basecamp). It is one of the fastest ways to go from idea to working application. The patterns you learn here (MVC, REST, CRUD) apply to every web framework.
