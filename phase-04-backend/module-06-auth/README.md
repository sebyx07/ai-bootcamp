# Module 06: Authentication

Authentication is how your app knows who is who. When someone logs in, how does the server remember them? How do you store passwords safely? This module covers the fundamentals of user authentication.

## What you will learn

- What authentication and authorization are
- How sessions and cookies work
- Why and how passwords are hashed
- How to build a login and signup flow
- What JWTs are and when they are used

## Challenges

| # | Challenge | What you will do |
|---|-----------|-----------------|
| 1 | What Is Auth | Understand authentication vs authorization |
| 2 | Sessions and Cookies | See how the server remembers logged-in users |
| 3 | Password Hashing | Learn why we never store plain passwords |
| 4 | Login/Signup Flow | Build actual signup and login functionality |
| 5 | JWT Basics | Understand JSON Web Tokens and when to use them |

## Why this matters

Almost every application needs users to log in. Getting authentication right is critical for security. Understanding how it works will make you a better developer and help you spot security issues.
