# Module 01: How the Web Works

Before you write any backend code, you need to understand what happens when you type a URL into your browser and press Enter. This module makes the invisible visible.

## What you will learn

- What HTTP requests and responses are
- How URLs work and what GET, POST, PUT, DELETE mean
- What JSON is and why it matters
- How to use curl and Postman to talk to servers directly

## Challenges

| # | Challenge | What you will do |
|---|-----------|-----------------|
| 1 | Request and Response | See the HTTP request/response cycle in action |
| 2 | URLs and HTTP Methods | Understand the parts of a URL and the main HTTP verbs |
| 3 | JSON | Read and write JSON, the language servers speak |
| 4 | Use curl and Postman | Talk to real APIs from the command line and Postman |

## Why this matters

Every single thing you build on the backend uses HTTP. Every API call, every form submission, every page load is a request and a response. If you understand this, everything else in this phase will make more sense.
