# Capstone Option A: Full-Stack Application

## The Project

Build a complete full-stack web application from scratch. This should be something you would actually use — not a tutorial clone.

## Requirements

### Must Have (MVP)
- A React frontend with multiple pages/views
- A backend API (Node.js/Express or Rails)
- A database (PostgreSQL or SQLite)
- User authentication (signup, login, logout)
- At least one CRUD resource (create, read, update, delete)
- Form validation (client and server side)
- Error handling throughout
- Deployed and accessible via URL

### Should Have
- Responsive design (works on mobile and desktop)
- Loading states and user feedback
- At least three database tables with relationships
- Search or filter functionality
- Tests for critical paths

### Nice to Have
- Real-time features (WebSockets)
- File uploads
- Email notifications
- Admin dashboard
- CI/CD pipeline

## Project Ideas

- **Task manager** with projects, tasks, due dates, and collaborators
- **Recipe sharing platform** with ingredients, categories, and user favorites
- **Personal finance tracker** with transactions, categories, and monthly reports
- **Book club app** with book lists, reviews, reading progress, and discussion threads
- **Event planner** with events, RSVPs, and calendar integration

You can also propose your own idea.

## Evaluation Criteria

- Does it work? Can someone sign up, use the core feature, and have a good experience?
- Is the code clean? Would a teammate be comfortable working in this codebase?
- Is it deployed? Can you share a link?
- Did you use the skills from this bootcamp? Git history, tests, CLAUDE.md, proper deployment?
