# Challenge: GitHub Actions Intro

## What you'll learn
How to set up a simple automated workflow that runs every time you push code — your first taste of CI/CD.

## Watch first
- [DevOps CI/CD Explained in 100 Seconds](https://www.youtube.com/watch?v=scEDHsr3APg) (2 min) — Fireship explains what CI/CD is and why it matters, fast
- [GitHub Actions Tutorial](https://www.youtube.com/watch?v=R8_veQiYBjI) (33 min) — TechWorld with Nana walks through GitHub Actions concepts and building a real workflow

## Open Claude Code and say:
> "I'm ready for the GitHub Actions challenge"

## You're done when
- [ ] You have a `.github/workflows/` directory in your repo with a YAML workflow file
- [ ] Your workflow runs automatically when you push to the repo
- [ ] You can see the green check or red X on your commit in GitHub
- [ ] You can open the Actions tab and read the workflow run log
- [ ] You can explain what CI stands for and why automated checks are useful
