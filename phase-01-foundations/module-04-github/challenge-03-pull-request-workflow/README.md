# Challenge: Pull Request Workflow

## What you'll learn
The complete pull request cycle — from creating a branch to merging it — the way professional developers ship code every day.

## Watch first
- [Git and GitHub for Beginners — Crash Course](https://www.youtube.com/watch?v=RGOj5yH7evk) (69 min) — Watch the pull request section starting around 50 minutes in for a clear walkthrough
- [Git and GitHub for Beginners Tutorial](https://www.youtube.com/watch?v=tRZGeaHPoaw) (46 min) — Kevin Stratvert covers the full PR flow including the GitHub UI

## Open Claude Code and say:
> "I'm ready for the pull request workflow challenge"

## You're done when
- [ ] You created a new branch, made changes, and pushed it to GitHub
- [ ] You opened a pull request with a clear title and description
- [ ] You can see the diff on the pull request page and explain what it shows
- [ ] You merged the pull request into main
- [ ] You can explain why teams use pull requests instead of pushing directly to main
