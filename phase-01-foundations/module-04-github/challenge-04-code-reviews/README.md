# Challenge: Code Reviews

## What you'll learn
How to review code on a pull request — reading diffs, leaving comments, and approving or requesting changes.

## Watch first
- [Git and GitHub for Beginners — Crash Course](https://www.youtube.com/watch?v=RGOj5yH7evk) (69 min) — Covers the collaboration workflow including reviews; watch from the pull request sections
- [Git and GitHub for Beginners Tutorial](https://www.youtube.com/watch?v=tRZGeaHPoaw) (46 min) — Kevin Stratvert shows the review process in the GitHub UI

## Open Claude Code and say:
> "I'm ready for the code reviews challenge"

## You're done when
- [ ] You have opened the "Files changed" tab on a pull request and read the diff
- [ ] You have left at least one line-level comment on a diff
- [ ] You have submitted a review using the "Review changes" button (comment, approve, or request changes)
- [ ] You can explain the difference between "Approve" and "Request changes"
- [ ] You can describe at least 3 things you would look for when reviewing someone else's code
