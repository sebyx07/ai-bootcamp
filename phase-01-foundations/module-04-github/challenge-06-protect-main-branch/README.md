# Challenge: Protect the Main Branch

## What you'll learn
How to set up branch protection rules so that no one — including you — can push directly to main without a pull request and passing checks.

## Watch first
- [GitHub Actions Tutorial](https://www.youtube.com/watch?v=eB0nUzAI7M8) (12 min) — Fireship covers GitHub Actions and CI/CD setup, which connects to why branch protection matters
- [DevOps CI/CD Explained in 100 Seconds](https://www.youtube.com/watch?v=scEDHsr3APg) (2 min) — Quick refresher on why automated checks and safe workflows matter

## Open Claude Code and say:
> "I'm ready for the branch protection challenge"

## You're done when
- [ ] You have set up a branch protection rule on your main branch
- [ ] The rule requires a pull request before merging
- [ ] You have tried pushing directly to main and seen it get rejected (or would be rejected)
- [ ] You can explain why teams protect their main branch
- [ ] You understand what "require status checks to pass" means
