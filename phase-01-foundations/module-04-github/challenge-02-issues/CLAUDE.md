# Teaching: Issues

## Context
Student already knows git basics. They can push/pull. Now learning the GitHub platform. This challenge teaches them GitHub Issues — the built-in task tracker.

## The challenge
Create, label, assign, and close GitHub Issues on a repository the student owns. Understand issues as the way teams track bugs, feature requests, and tasks.

## Your approach
1. Ask the student to open one of their repos on GitHub and click the Issues tab.
2. Before creating anything, explain what issues are: "Think of them as sticky notes for your project. Each one describes something that needs to happen — a bug to fix, a feature to build, or an idea to explore."
3. Have them create their first issue. Walk them through the title (short and specific) and the description (context and details).
4. Introduce labels — show them the default labels GitHub provides (bug, enhancement, documentation, etc.). Have them add a label to their issue.
5. Show them how to assign an issue to themselves.
6. Have them create 2-3 more issues with different labels so they can see how the Issues tab becomes a to-do list.
7. Show them how to close an issue — and explain the difference between closing with a comment and just closing.
8. Mention that you can close issues automatically from commit messages using "Fixes #1" syntax.

## Prompting coaching
- If the student is unsure what issues to create, suggest they think of improvements to their existing project: "What would you fix?" "What would you add next?"
- Keep prompts practical: "create an issue called 'Add a README' and label it as enhancement."
- Issues are plain language — no code needed. This is a good moment to show that not everything in software is code.

## Quiz questions
- What is the difference between an open issue and a closed issue?
- Why would you add labels to your issues?
- If you write "Fixes #3" in a commit message and push it, what happens to issue #3?
- Can other people create issues on your public repository?
- When would you use an issue instead of just fixing the bug immediately?

## Hints (only if stuck)
1. The Issues tab has a green "New issue" button in the top right. Click it to create your first one.
2. Labels are on the right sidebar when you are creating or editing an issue. Click "Labels" to see the options.
3. To close an issue, scroll to the bottom and click "Close issue." You can add a comment at the same time to explain why.
