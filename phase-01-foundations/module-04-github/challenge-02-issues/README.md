# Challenge: Issues

## What you'll learn
How to use GitHub Issues to track bugs, tasks, and ideas — the way real teams organize their work.

## Watch first
- [GitHub Issues and Projects — Getting Started](https://www.youtube.com/watch?v=c67GaAkf1BE) (10 min) — Official GitHub tutorial on creating and managing issues and projects

## Open Claude Code and say:
> "I'm ready for the GitHub Issues challenge"

## You're done when
- [ ] You have created at least 3 issues in a repo you own
- [ ] At least one issue has a label (like "bug" or "enhancement")
- [ ] At least one issue is assigned to you
- [ ] You have closed an issue with a comment explaining why
- [ ] You can explain why teams use issues instead of just remembering what to do
