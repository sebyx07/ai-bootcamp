# Module 04: GitHub

You already know Git. You can init, add, commit, branch, merge, push, and pull. Now it is time to learn the platform those commands talk to: GitHub.

GitHub is not Git. Git is the version control tool that runs on your computer. GitHub is the website where your code lives online, where teams collaborate, where pull requests get reviewed, and where automation runs your tests every time you push.

This module is about the GitHub website itself — what all those tabs mean, how teams actually use it day to day, and how to set up guardrails that keep your code safe.

## Challenges

1. **The GitHub Interface** — Learn what you are looking at on a repo page
2. **Issues** — Track work, bugs, and ideas with GitHub Issues
3. **Pull Request Workflow** — The full cycle from branch to merge
4. **Code Reviews** — Read diffs, leave comments, approve or request changes
5. **GitHub Actions Intro** — Automated checks that run when you push
6. **Protect the Main Branch** — Rules that keep bad code out of production
