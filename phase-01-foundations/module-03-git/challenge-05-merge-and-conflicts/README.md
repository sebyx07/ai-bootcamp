# Challenge: Merge and Conflicts

## What you'll learn
How to bring branches back together with `git merge`, and what to do when two branches changed the same thing.

## Watch first
- [Git & GitHub Crash Course 2025](https://www.youtube.com/watch?v=vA5TTz6BXhY) (watch minutes 35-45) — Traversy Media covers merging and conflict resolution
- [Git and GitHub for Beginners — Crash Course](https://www.youtube.com/watch?v=RGOj5yH7evk) (watch minutes 45-55) — freeCodeCamp walks through a real merge conflict

## Open Claude Code and say:
> "I'm ready for the challenge"

## You're done when
- [ ] You merged one branch into another with `git merge`
- [ ] You created a merge conflict on purpose
- [ ] You resolved the conflict by editing the file
- [ ] You understand what the conflict markers (`<<<<<<<`, `=======`, `>>>>>>>`) mean
