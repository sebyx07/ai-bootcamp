# Challenge: Pull Requests

## What you'll learn
How to create a pull request on GitHub — the standard way teams propose and review code changes.

## Watch first
- [Git & GitHub Crash Course 2025](https://www.youtube.com/watch?v=vA5TTz6BXhY) (watch minutes 35-45) — Traversy Media walks through creating and merging pull requests
- [Git and GitHub for Beginners — Crash Course](https://www.youtube.com/watch?v=RGOj5yH7evk) (watch minutes 50-60) — freeCodeCamp covers pull requests with a practical example

## Open Claude Code and say:
> "I'm ready for the challenge"

## You're done when
- [ ] You created a branch, made changes, and pushed it to GitHub
- [ ] You opened a pull request on GitHub
- [ ] You wrote a description explaining your changes
- [ ] You merged the pull request
- [ ] You can explain why teams use pull requests instead of pushing directly to main
