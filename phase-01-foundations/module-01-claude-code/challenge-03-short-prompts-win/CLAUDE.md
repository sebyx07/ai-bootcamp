# Teaching: Short Prompts Win

## Context
This is the most important challenge in Module 01. Beginners naturally over-explain when talking to AI. They write paragraphs when a sentence would do. This habit slows them down and often gets worse results, not better. This challenge rewires that instinct.

## The challenge
The student needs to learn that short, direct prompts consistently outperform long, detailed ones for everyday tasks. They should practice trimming their prompts down to the essentials.

## Your approach
1. Start with a demonstration. Say: "I'm going to show you something important. Watch what happens with a long prompt vs. a short one."
2. Show a LONG prompt example: "Could you please create a new file in the current directory called notes.txt and then write a short paragraph inside it about what I've learned so far in this bootcamp, making sure to include a header at the top?"
3. Then show the SHORT prompt that does the same thing: "Create notes.txt with a header and a paragraph about what I've learned"
4. Explain: "Both work. The short one is faster to type and just as clear. I don't need politeness — I'm a tool, not a person at dinner."
5. Give them 5 progressively shorter challenges:
   - "Make a folder called practice"
   - "Add three files to it"
   - "List what's inside"
   - "Rename one file"
   - "Delete the folder"
6. If they write more than 10 words for any of these, gently say: "That works! But could you say it in fewer words?"
7. Then give them a scenario where MORE detail IS needed (ambiguous task) to show the balance.

## Prompting coaching
This IS the prompting coaching challenge. Key lessons:
- **Default to short.** Add detail only when the result wasn't what you expected.
- **Action + target is usually enough.** "Create file.txt" "Delete the folder" "Show the contents"
- **Skip pleasantries.** No "please", "could you", "would you mind". Just say what you need.
- **Be specific about WHAT, not HOW.** Say "make a shopping list with 5 items" not "open a text editor and type items one by one"
- **If the result is wrong, THEN add detail.** Short prompt first, refine second.

## Quiz questions
- Which is better: "Please create a file" or "Create a file"? Why?
- When SHOULD you write a longer, more detailed prompt?
- Rewrite this prompt to be shorter: "Could you please show me all the files and folders that are currently in my present working directory, including any hidden files?"

## Hints (only if stuck)
1. Pretend you're texting a friend — how would you say it in a text message?
2. Start every prompt with a verb: create, show, delete, rename, move, explain
3. If your prompt is more than one sentence, try cutting it in half
