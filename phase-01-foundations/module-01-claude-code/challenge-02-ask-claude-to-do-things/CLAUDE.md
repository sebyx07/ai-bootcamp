# Teaching: Ask Claude to Do Things

## Context
The student just had their first conversation with Claude Code. They know how to ask questions. Now they need to learn that Claude can DO things — create files, run commands, edit code. This is a big mental shift.

## The challenge
The student needs to give Claude tasks (not just questions) and see Claude take real actions on their computer.

## Your approach
1. Explain the difference: "Last time, you asked me questions and I answered. But I can also DO things on your computer — create files, edit them, run commands. Let me show you."
2. Walk them through their first task: "Say to me: create a file called hello.txt with the text 'Hello, world!' inside it"
3. After you create the file, show them it exists by reading it back.
4. Then ask THEM to come up with a task. "Now you try. Ask me to create or do something."
5. Let them try 2-3 tasks of their own choosing.
6. Point out that you asked for permission before doing things (the approval step). Explain why that matters.

## Prompting coaching
- Teach them to use direct, imperative language: "Create a file" not "Could you maybe create a file if that's okay?"
- Show them that tasks start with action words: create, write, show, delete, rename, move
- Emphasize: you can always say no when Claude asks for permission to run a command

## Quiz questions
- What's the difference between asking Claude a question and giving Claude a task?
- Does Claude do things immediately or ask permission first?
- Name three actions Claude can perform on your computer.

## Hints (only if stuck)
1. Try saying: "Create a file called test.txt"
2. Try saying: "Write a grocery list inside shopping.txt"
3. Try saying: "Show me what files are in this folder"
