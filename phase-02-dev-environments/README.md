# Phase 02: Dev Environments

## What this phase is about

Your computer needs specific tools installed before you can build software. This phase turns your machine into a real development environment -- the same kind professional developers use every day.

By the end of this phase, you will have package managers, programming languages, Docker, and databases all running on your machine. More importantly, you will know how to fix things when they inevitably break.

## Modules

| Module | What you will learn |
|--------|-------------------|
| **01 - Package Managers** | How software gets installed on Linux, and why you never download .exe files |
| **02 - Node and NVM** | Install JavaScript's runtime and learn to switch between versions |
| **03 - Ruby and rbenv** | Install Ruby and understand how gems (libraries) work |
| **04 - Docker** | Run entire applications inside containers -- the most important dev tool of the decade |
| **05 - Databases** | Install PostgreSQL and Redis so your apps can store data |
| **06 - Things Will Break** | The most important module -- learn to diagnose and fix real errors |

## Prerequisites

- Completed Phase 01 (you should be comfortable with the terminal)
- A Linux-based system (native Linux, WSL on Windows, or macOS)

## How long will this take?

Expect 8-12 hours spread across several days. Module 04 (Docker) is the longest. Module 06 (Things Will Break) is the most valuable -- do not skip it.

## Tips for this phase

- **Read error messages.** They look scary but they almost always tell you what went wrong.
- **Do not copy-paste commands you do not understand.** Ask Claude Code to explain first.
- **Things will break.** That is normal. Module 06 exists specifically because breaking and fixing things is how you actually learn.
- **Take notes on what you install.** Future-you will thank present-you.
