# Module 06: Things Will Break

## What you will learn

How to diagnose and fix the most common development environment errors. This is arguably the most important module in this phase.

## Why this matters

Professional developers spend a huge amount of time debugging environment issues. The difference between a beginner and an intermediate developer is not that intermediates never see errors -- it is that they know how to read them and fix them. This module gives you that skill.

## Challenges

| Challenge | Topic |
|-----------|-------|
| 01 | Command not found -- the most common beginner error |
| 02 | Permission denied -- understanding file permissions and sudo |
| 03 | Port already in use -- when two programs fight over the same port |
| 04 | Wrong version running -- when the wrong version of a tool is active |
| 05 | PATH issues -- the invisible variable that controls everything |

## How this module works

Each challenge intentionally creates a broken scenario. Your job is to figure out what went wrong and fix it. Claude Code will set up the problem and guide you through diagnosing it -- but you have to do the detective work.

## Key concepts

- **Error messages are your friends.** They tell you exactly what went wrong if you learn to read them.
- **Google the error message.** This is literally what professional developers do.
- **Check the obvious things first.** Is the service running? Is the path correct? Do you have permission?
- **Undo what you just did.** If something broke after you changed something, start by reverting that change.
