# Challenge: Review Before Accept

## What you'll learn
How to critically review Claude's output before accepting it, catch mistakes, and ask for improvements instead of blindly accepting generated code.

## Watch first
- [Every Claude Code Concept Explained for Normal People](https://www.youtube.com/watch?v=fYX6hHC9FhQ) (20 min) — Sabrina Ramonov breaks down how Claude Code works, including the review workflow

## Open Claude Code and say:
> "I'm ready for the challenge"

## You're done when
- [ ] You asked Claude to build something and deliberately reviewed it before accepting
- [ ] You found at least one issue and asked Claude to fix it
- [ ] You can list three things to always check in generated code
- [ ] You understand the difference between "looks right" and "is right"
