# Phase 07: Advanced Claude Code — Teaching Notes

## Who you're teaching

Students who have completed six phases of the bootcamp. They can write code, use the terminal, manage Git, build frontend and backend projects, write tests, and deploy. They are comfortable with Claude Code but have only scratched the surface. Now they need to become power users.

## Teaching principles for this phase

1. **Show the difference.** For every technique, show the "before" (how they'd do it now) and the "after" (the power-user way). The contrast teaches.
2. **Make them configure real projects.** Abstract examples don't stick. Every CLAUDE.md they write should be for a project they care about.
3. **Let them feel the speed.** When subagents or hooks save time, point it out. "That took 10 seconds instead of 10 minutes."
4. **Encourage experimentation.** There is no single right way to prompt or configure. Encourage them to try variations and see what works.
5. **Treat mistakes as data.** When a prompt produces bad output, that is a learning opportunity. Ask "why do you think it gave you that?" before fixing it.

## Pacing

- This phase moves faster than earlier phases. Students should be comfortable with trial and error by now.
- Module 05 (AI-First Project) is the largest — expect it to take as long as the other four modules combined.
- If a student already has strong prompting instincts, let them skim Module 01 and spend more time on Modules 03-04.

## Common mistakes at this level

- Writing CLAUDE.md files that are too vague ("write good code") or too rigid (listing every file path)
- Over-prompting: giving Claude step-by-step instructions instead of context and goals
- Not reviewing AI output before accepting — trusting too much
- Ignoring hooks and skills because they seem "advanced" — they are simpler than they look
- Treating MCP servers as magic instead of understanding what they connect to

## Module flow

Module 01 levels up their prompting. Module 02 teaches them to configure Claude for specific projects. Module 03 adds automation. Module 04 expands what Claude can do with subagents and external tools. Module 05 ties it all together in a real project.
