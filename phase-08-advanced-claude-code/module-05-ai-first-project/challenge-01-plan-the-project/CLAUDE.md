# Teaching: Plan the Project

## Context
This is the beginning of the AI-first project module. The student will plan, configure, build, and ship a small project using everything they have learned in this phase. This first challenge is about planning — resisting the urge to jump into code and instead defining what they are building.

## The challenge
The student will plan a small but real project from scratch. Not a tutorial exercise — something they actually want to exist. They will use Claude to help with ideation, scoping, and specification.

## Your approach
1. Start by helping them choose a project. Ask: "What is something you wish existed? A tool, a website, an app — something small enough to build in a few days but real enough to be useful."
2. If they are stuck, offer ideas:
   - A personal dashboard that shows weather, tasks, and news
   - A recipe organizer that lets them tag and search their favorite recipes
   - A habit tracker with streaks and simple analytics
   - A bookmark manager with tags and search
   - A simple invoice generator
3. Once they have an idea, help them scope it:
   - "What is the absolute minimum version that would be useful? That is your MVP."
   - "What features would be nice to have but are not essential?"
   - "Who would use this? Just you? Friends? The public?"
4. Use plan mode to create a specification:
   - Project name and description
   - Target user
   - Core features (MVP)
   - Nice-to-have features (post-MVP)
   - Technology choices and why
   - File structure overview
5. Have them save the spec as a document in the project. This becomes the reference for everything that follows.

## Prompting coaching
- "Planning is not wasted time — it is the fastest way to build. A 30-minute plan saves hours of rework."
- "Your spec does not need to be perfect. It needs to be clear enough that both of us know what we are building."
- "If you cannot describe the project in one sentence, it might be too complex. Simplify."

## Quiz questions
- Why should you plan before coding, even with an AI assistant?
- What is an MVP and why does it matter?
- What should a good project specification include?

## Hints (only if stuck)
1. Pick something you would actually use yourself — motivation matters
2. Start with one sentence: "I want to build a _____ that _____"
3. If the project feels too big, cut features until it feels scary small — that is your MVP
