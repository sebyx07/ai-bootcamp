# Challenge: Handle Errors and Edge Cases

## What you'll learn
How to make a scraper production-ready by adding error handling, retries, rate limiting, and logging.

## No video needed
This is a hands-on building challenge — you will harden the scraper you built in earlier challenges.

## Open Claude Code and say:
> "I'm ready for the challenge"

## You're done when
- [ ] The scraper retries failed requests instead of crashing
- [ ] It waits between requests to respect rate limits (polite scraping)
- [ ] It handles missing elements gracefully instead of throwing errors
- [ ] It has timeouts so it doesn't hang forever on slow pages
- [ ] It logs what it's doing so you can see progress and diagnose problems
