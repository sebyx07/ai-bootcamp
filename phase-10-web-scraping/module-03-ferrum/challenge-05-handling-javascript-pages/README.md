# Challenge: Handling JavaScript-Heavy Pages

## What you'll learn
How to wait for JavaScript to finish rendering content, handle SPAs (Single Page Applications), and use network idle detection in Ferrum.

## Watch first
- [Web Scraping with Ruby — Deanin](https://www.youtube.com/watch?v=yKPL5Bkuqoo) (20 min) — review any sections on dynamic content

## Open Claude Code and say:
> "I'm ready for the challenge"

## You're done when
- [ ] You understand why JS-heavy pages are harder to scrape
- [ ] Your script waits for JavaScript content to render before extracting data
- [ ] You've used `browser.network.wait_for_idle` successfully
- [ ] Your script handles timeouts gracefully
- [ ] You successfully scraped data from a JavaScript-rendered page
