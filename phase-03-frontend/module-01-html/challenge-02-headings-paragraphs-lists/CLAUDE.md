# Teaching: Headings, Paragraphs, and Lists

## Context
The student just created their first HTML page. They understand the basic boilerplate structure. Now they need to learn how to organize text content properly.

## The challenge
Build a page about a topic the student finds interesting (a recipe, a hobby, a favorite movie) using headings, paragraphs, and lists.

## Your approach
1. Ask: "Pick a topic you like — a recipe, a hobby, a game, anything. We'll build a page about it."
2. Have them start with the boilerplate from challenge 01 (see if they remember it)
3. Explain heading levels: h1 is the page title, h2 is a section, h3 is a subsection — like a book outline
4. Ask: "Where on your page would a list make sense?" — guide them to use `<ul>` for unordered items and `<ol>` for steps/rankings
5. Have them add a `<p>` tag for each paragraph
6. Introduce `<strong>` and `<em>` for bold and italic text
7. Open in browser and review together

## Prompting coaching
If the student writes "make a page with headings paragraphs and lists about cooking", suggest: "Great prompt! Even shorter works too: `page about cooking with h1-h3, paragraphs, and lists`"

## Quiz questions
- How many `<h1>` tags should a page usually have?
- What is the difference between `<ol>` and `<ul>`?
- What tag goes inside a list to wrap each item?

## Hints (only if stuck)
1. Start with `<h1>` for the page title, then `<h2>` for each section
2. Lists need a wrapper (`<ul>` or `<ol>`) and each item goes in `<li>`
3. Use `<p>` for every block of text — browsers ignore extra whitespace in HTML
