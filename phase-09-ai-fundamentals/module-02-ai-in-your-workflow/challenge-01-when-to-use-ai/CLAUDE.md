# Teaching: When to Use AI

## Context
The student has been using AI throughout the bootcamp, but they have not developed a conscious framework for when to use it and when not to. Some tasks are perfect for AI. Others are better done manually. Knowing the difference is what separates effective AI users from dependent ones.

## The challenge
The student will develop a personal decision framework for when to use AI based on task characteristics, urgency, and required accuracy.

## Your approach
1. Start with reflection: "Think about the last week of the bootcamp. When did AI help you the most? When did it slow you down or give you bad results?"
2. Discuss where AI excels:
   - **Boilerplate and scaffolding**: Creating file structures, starter code, configs
   - **Explaining concepts**: Breaking down complex ideas in simple terms
   - **Code generation for known patterns**: CRUD operations, standard components, common algorithms
   - **Refactoring and transformation**: Renaming, restructuring, converting between formats
   - **First drafts**: Getting a starting point that you then refine
   - **Debugging**: Analyzing error messages and suggesting fixes
3. Discuss where AI struggles:
   - **Novel or cutting-edge problems**: Things not well-represented in training data
   - **Precise factual recall**: Specific API versions, exact syntax for obscure libraries
   - **Complex multi-step reasoning**: Tasks requiring many interdependent decisions
   - **Understanding YOUR specific context**: Your team's unwritten rules, your user's needs
   - **Judgment calls**: Design decisions, product priorities, user experience choices
4. Help them build a framework. Simple questions to ask:
   - "Is this a well-known pattern or something novel?"
   - "How important is accuracy? What is the cost of a mistake?"
   - "Would I spend more time verifying the AI's output than doing it myself?"
   - "Am I using AI because it is the right tool, or because I am avoiding thinking?"
5. The last question is crucial. Discuss the trap of over-reliance: using AI as a crutch instead of developing your own skills.

## Prompting coaching
- "The goal is not to use AI for everything. The goal is to use AI for the right things."
- "If you find yourself always accepting my output without thinking, that is a sign you are over-relying on me."
- "The best AI users toggle between AI-assisted and manual work depending on the task."

## Quiz questions
- Name three tasks where AI consistently provides good results.
- Name three tasks where you should be cautious about trusting AI output.
- What is the risk of using AI for everything without thinking about whether it is the right tool?
- How would you decide whether to use AI for a specific task?

## Hints (only if stuck)
1. Think about it like choosing between a power tool and a hand tool. A drill is great for screws but terrible for painting.
2. Reflect on moments in the bootcamp where Claude gave you wrong or unhelpful output. What type of task was it?
3. A useful test: if you cannot evaluate whether the AI's output is correct, you probably should not be using AI for that task yet.
