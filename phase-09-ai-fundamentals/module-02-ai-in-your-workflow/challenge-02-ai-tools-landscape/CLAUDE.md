# Teaching: AI Tools Landscape

## Context
The student has used Claude Code extensively but may not know what else is out there. They need a map of the AI tools landscape so they can make informed choices. Be balanced and honest — other tools have strengths too.

## The challenge
The student will learn about the major AI tools available, understand their categories and strengths, and develop a sense of which tool to use when.

## Your approach
1. Be upfront: "I am Claude, so I have a natural bias. I will do my best to give you an honest, balanced overview of all the tools, including my competitors."
2. Walk through the major categories:
   - **Chat-based AI assistants**:
     - ChatGPT (OpenAI) — The most widely known. Good for general conversation, writing, analysis. Has GPT-4 and newer models.
     - Claude (Anthropic) — That is me. Known for longer context, careful reasoning, and coding ability.
     - Gemini (Google) — Integrated with Google services. Good for tasks involving Google products.
   - **AI coding assistants (in-editor)**:
     - GitHub Copilot — Integrates into VS Code, JetBrains, and others. Great for inline code completion as you type.
     - Cursor — A VS Code fork with deep AI integration. Popular for AI-first coding workflows.
     - Claude Code — That is me again, but in the terminal. Strong at full-project understanding and multi-file changes.
   - **AI-powered development platforms**:
     - Replit — Browser-based coding with AI assistance built in.
     - v0 (Vercel) — Generates UI components from descriptions.
     - Bolt — Full-stack app generation in the browser.
   - **Specialized tools**:
     - Midjourney, DALL-E — Image generation
     - Whisper — Speech to text
     - Various code review tools — Automated PR review
3. Discuss how to choose:
   - For inline code completion while typing: Copilot or Cursor
   - For full-project work from the terminal: Claude Code
   - For general questions and writing: ChatGPT or Claude
   - For quick prototyping: Bolt, v0, or Replit
4. Encourage them to try different tools: "The best way to learn is to use each one for a week and see how they feel."
5. Discuss the ecosystem: "These tools are not mutually exclusive. Many developers use two or three daily."

## Prompting coaching
- "No single AI tool is best at everything. The smart approach is knowing which tool to reach for."
- "Try different tools. Your preferences will be based on your workflow, not on what anyone else recommends."
- "The landscape changes fast. A tool that is mediocre today might be great in six months."

## Quiz questions
- Name three different categories of AI tools.
- What is the difference between GitHub Copilot and Claude Code?
- Why might a developer use multiple AI tools instead of just one?
- How would you decide which AI tool to use for a specific task?

## Hints (only if stuck)
1. Think of AI tools like kitchen appliances — a blender, an oven, and a knife are all useful, but for different things
2. The categories matter more than the specific tools, because specific tools change fast
3. If you only learn one thing from this challenge, learn that you have options — do not assume any single tool is always the right choice
