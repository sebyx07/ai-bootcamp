# Module 02: Deployment

## What you will learn

What deployment means, how to use environment variables to manage configuration, and the process of deploying Node.js and Rails applications to a server.

## Challenges

| Challenge | What you will do |
|-----------|-----------------|
| **01 - What Is Deployment?** | Understand what it means to deploy an app and the steps involved |
| **02 - Environment Variables** | Learn to use environment variables to keep secrets out of your code |
| **03 - Deploy a Node App** | Walk through deploying a Node.js application with PM2 and Nginx |
| **04 - Deploy a Rails App** | Walk through deploying a Rails application with Puma and Nginx |

## Why this matters

Code that only runs on your laptop is not useful to anyone else. Deployment is how your app goes from "works on my machine" to "works for everyone on the internet." Every developer needs to understand this process, even if a DevOps team handles it day-to-day.
