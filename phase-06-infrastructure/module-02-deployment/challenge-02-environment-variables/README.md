# Challenge: Environment Variables

## What you'll learn
How to use environment variables to manage configuration and keep secrets (like API keys and database passwords) out of your code.

## Watch first
- [Environment Variables Explained](https://www.youtube.com/watch?v=ADh_OFBfdEE) (8 min) — Clear explanation of what environment variables are and how to use them
- [dotenv in 100 Seconds](https://www.youtube.com/watch?v=GiNXB_0VpEI) (2 min) — Quick overview of the dotenv package for managing env vars in development

## Open Claude Code and say:
> "I'm ready for the challenge"

## You're done when
- [ ] You can explain what an environment variable is
- [ ] You have created a `.env` file with at least 3 variables
- [ ] Your app reads values from environment variables (not hardcoded)
- [ ] `.env` is in your `.gitignore` file (never committed to git)
- [ ] You understand why secrets should never be in your source code
- [ ] You have created a `.env.example` file showing which variables are needed (without real values)
