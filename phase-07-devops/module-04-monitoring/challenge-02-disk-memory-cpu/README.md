# Challenge: Disk, Memory, CPU

## What you'll learn
How to monitor system resources — disk space, memory usage, and CPU load — using htop, df, and free.

## Watch first
- [Linux htop Explained](https://www.youtube.com/watch?v=tIUEbRPpbSg) (10 min) — Veronica Explains walks through htop and what all the numbers mean
- [Linux Disk Space Commands - df and du](https://www.youtube.com/watch?v=wkGNBBbfvbk) (8 min) — Learn Linux TV explains how to check disk usage

## Open Claude Code and say:
> "I'm ready for the challenge"

## You're done when
- [ ] You can check disk space with `df -h` and explain what each column means
- [ ] You can check memory with `free -h` and explain used vs available
- [ ] You can use `htop` to see running processes sorted by CPU or memory usage
- [ ] You can identify which process is using the most CPU or memory
- [ ] You know the warning signs: disk over 90%, memory swap being used heavily, CPU consistently at 100%
