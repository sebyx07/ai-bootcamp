# Module 04: Monitoring

## What this module is about

A running server is not a finished job. You need to know if it is healthy, if it is running out of resources, and what is going wrong when something breaks. This module teaches you the essential monitoring tools that every system administrator uses daily.

## Challenges

| Challenge | What you will do |
|-----------|-----------------|
| **01 - Check Server Health** | Learn the first commands to run when you SSH into a server |
| **02 - Disk, Memory, CPU** | Monitor system resources with htop, df, and free |
| **03 - Reading Logs** | Use journalctl and log files to debug problems |

## By the end of this module

You will know how to quickly assess whether a server is healthy, identify resource problems, and find error messages in logs. These are the skills you use when something goes wrong at 2 AM.
