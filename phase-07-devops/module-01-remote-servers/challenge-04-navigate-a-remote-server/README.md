# Challenge: Navigate a Remote Server

## What you'll learn
How to move around a remote server's filesystem, find important directories, and read files — all over SSH.

## Watch first
- [Linux File System Explained](https://www.youtube.com/watch?v=HbgzrKJvDRw) (16 min) — DorianDotSlash walks through the Linux directory structure so you know what lives where

## Open Claude Code and say:
> "I'm ready for the challenge"

## You're done when
- [ ] You can navigate to key directories: `/etc`, `/var/log`, `/home`, `/tmp`, `/usr`
- [ ] You can explain what at least three of those directories are for
- [ ] You can find and read files using `cat`, `less`, and `grep` on the remote server
- [ ] You can use `find` to locate files on the server
- [ ] You are comfortable using `nano` to edit files on the remote server
