# Capstone Option B: SaaS MVP

## The Project

Build a minimum viable product for a software-as-a-service (SaaS) application. This is not just a coding exercise — it is a product exercise. You need to think about what problem you are solving, who your user is, and what the smallest version of the product looks like.

## Requirements

### Must Have (MVP)
- A compelling landing page that explains the product
- User authentication (signup, login, logout)
- One core feature that delivers value (the reason someone would pay)
- A clean, professional UI
- Deployed and accessible via URL

### Should Have
- Responsive design
- User onboarding flow (first-time user experience)
- Settings or profile page
- Usage limits or tiers (even if not enforced — show the concept)
- Error handling and loading states

### Nice to Have
- Stripe integration for payments (test mode is fine)
- Usage analytics or dashboard
- Email notifications
- Team or multi-user support
- Custom domain

## Project Ideas

- **Invoice generator** — Create, send, and track invoices. Core feature: generate a professional PDF invoice.
- **Link shortener with analytics** — Shorten URLs and track clicks, referrers, and locations.
- **Appointment scheduler** — Share a booking link, let people pick available times.
- **Feedback collector** — Embed a widget on any site to collect user feedback. Dashboard to review.
- **Content calendar** — Plan and schedule social media posts across platforms.

You can also propose your own SaaS idea.

## Evaluation Criteria

- Would someone actually use this? Is the core feature genuinely useful?
- Does the landing page clearly explain what the product does?
- Is the user experience smooth from landing page to core feature?
- Is the code production-ready? Authentication secure, errors handled, inputs validated?
- Is it deployed and shareable?
