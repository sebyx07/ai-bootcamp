# Phase 05: Testing & Quality

## What this phase is about

You have built real things in the previous phases -- websites, APIs, databases. But how do you know they actually work? How do you know they still work after you change something? Testing is how professional developers sleep at night. This phase teaches you to write automated tests, lint your code for consistency, and set up pipelines that catch problems before they reach users.

By the end of this phase, you will understand why testing matters, write tests in both JavaScript and Ruby, use linters to enforce code style, and set up a basic CI/CD pipeline with GitHub Actions.

## Modules

| Module | What you will learn |
|--------|-------------------|
| **01 - Why Testing** | What a test actually is, and why automated tests beat manual checking |
| **02 - JavaScript Testing** | Write your first Jest tests for functions and API endpoints |
| **03 - Ruby Testing** | Write your first RSpec tests for models and controllers |
| **04 - Linting** | Use ESLint and RuboCop to keep your code clean and consistent |
| **05 - CI/CD** | Automate testing with GitHub Actions so every push gets checked |

## Prerequisites

- Completed Phase 03 (you should be comfortable writing JavaScript)
- Completed Phase 04 (you should have built Node.js and Rails apps)
- A GitHub account with at least one repository

## How long will this take?

Expect 6-10 hours. Modules 02 and 03 (writing actual tests) take the longest. Module 05 (CI/CD) is the most satisfying -- watching your tests run automatically is a great feeling.

## Tips for this phase

- **Testing is not optional.** Every professional codebase has tests. Learn to love them now.
- **Start small.** Your first test will feel silly. That is fine. Complexity comes later.
- **Red, green, refactor.** Write a failing test, make it pass, then clean up. This rhythm will become second nature.
- **Linting is not nitpicking.** Consistent code is readable code, and readable code has fewer bugs.
- **CI/CD is your safety net.** Once it is set up, it catches mistakes you would have missed.
