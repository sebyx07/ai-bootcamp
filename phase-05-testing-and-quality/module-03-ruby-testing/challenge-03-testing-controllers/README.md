# Challenge: Testing Controllers

## What you'll learn
How to write RSpec tests for Rails controller actions to verify they return the right responses and status codes.

## Watch first
- [Rails Controller Testing with RSpec](https://www.youtube.com/watch?v=rHJmGrGkb3Q) (18 min) — Step-by-step guide to testing Rails controllers

## Open Claude Code and say:
> "I'm ready for the challenge"

## You're done when
- [ ] You have a Rails controller with at least `index`, `show`, and `create` actions
- [ ] You have request specs (not controller specs) that test each action
- [ ] Your tests verify status codes (200, 201, 404, 422)
- [ ] Your tests verify response body content
- [ ] Running `bundle exec rspec` shows all tests passing
