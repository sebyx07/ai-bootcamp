# Module 01: Why Testing

## What you will learn

What software testing actually is, why developers write tests, and the difference between testing manually and using automated tests.

## Challenges

| Challenge | What you will do |
|-----------|-----------------|
| **01 - What Is a Test?** | Understand what a test is by writing one in plain English, then in code |
| **02 - Manual vs Automated** | Experience manual testing, then see how automation saves time and catches bugs |

## Why this matters

Every piece of professional software has tests. Without them, every change you make could secretly break something else. Testing is not extra work -- it is the work that prevents future disasters.
