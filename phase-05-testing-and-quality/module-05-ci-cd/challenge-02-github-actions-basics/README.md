# Challenge: GitHub Actions Basics

## What you'll learn
How to create your first GitHub Actions workflow file that runs automatically when you push code.

## Watch first
- [GitHub Actions - Supercharged CI/CD](https://www.youtube.com/watch?v=eB0nUzAI7M8) (12 min) — Fireship's practical tutorial on setting up GitHub Actions from scratch
- [GitHub Actions Tutorial for Beginners](https://www.youtube.com/watch?v=mFFXuXjVgkU) (33 min) — TechWorld with Nana's step-by-step beginner guide

## Open Claude Code and say:
> "I'm ready for the challenge"

## You're done when
- [ ] You have a `.github/workflows/` directory in your repository
- [ ] You have created a workflow YAML file that triggers on push
- [ ] The workflow runs at least one step (even if it is just `echo "Hello CI!"`)
- [ ] You have pushed the workflow to GitHub and seen it run in the Actions tab
- [ ] You understand the structure: workflow > jobs > steps
