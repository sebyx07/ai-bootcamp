# Module 05: CI/CD

## What you will learn

What Continuous Integration and Continuous Deployment mean, how GitHub Actions works, and how to set up a pipeline that automatically runs your tests every time you push code.

## Challenges

| Challenge | What you will do |
|-----------|-----------------|
| **01 - What Is CI/CD?** | Understand the concepts of continuous integration and continuous deployment |
| **02 - GitHub Actions Basics** | Create your first GitHub Actions workflow file |
| **03 - Auto-Run Tests on Push** | Set up a pipeline that runs your test suite every time you push to GitHub |

## Why this matters

CI/CD is how modern teams ship software safely. Instead of hoping everything works, you set up automated checks that run on every change. If something breaks, you know immediately -- before it reaches users.
